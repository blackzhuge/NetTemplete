# 脚手架配置器增强：预设系统 + 代码预览

## 概述

增强脚手架配置器的交互体验，支持预设模板快速配置，并提供点击文件预览代码的功能。

## 背景

当前脚手架配置器功能单一：
- 用户需要手动逐项配置，无法快速选择常用组合
- 文件树仅展示结构，无法预览生成的代码内容
- 用户需要下载 ZIP 后才能查看实际生成的代码

## 目标

1. **预设模板系统** - 从后端 API 获取预设列表，用户选择后自动填充配置
2. **交互式文件预览** - 选中文件树节点后，侧边栏实时显示渲染后的代码
3. **代码语法高亮** - 使用 Shiki 实现 C#/Vue/TypeScript/JSON 等语言高亮

## 非目标

- 在线编辑生成的代码
- 用户自定义预设保存
- 文件对比功能

## 技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 高亮库 | Shiki | VS Code 同款引擎，准确度高，SSR 友好 |
| 预设来源 | 后端 API | 支持动态更新，前后端一致性 |
| 预览触发 | 选中文件节点 | 明确的用户意图，避免频繁请求 |
| 预览来源 | 后端实时渲染 | 保证与生成结果完全一致 |

## 用户故事

1. 作为开发者，我想选择一个预设模板，快速填充常用配置组合
2. 作为开发者，我想点击文件树中的文件，预览即将生成的代码内容
3. 作为开发者，我想看到语法高亮的代码，以便更容易理解生成结果
