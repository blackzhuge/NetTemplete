# 配置器使用说明

本文档介绍如何使用 Scaffold Generator 的 Web 配置器生成项目。

## 访问配置器

启动开发服务器后，访问 `http://localhost:5173` 打开配置器界面。

## 界面布局

配置器采用左右分栏布局：

- **左侧**: 配置表单
- **右侧**: 文件结构预览

## 配置选项

### 基本配置

| 选项 | 说明 | 示例 |
|------|------|------|
| 项目名称 | 生成项目的名称 | `MyProject` |
| 命名空间 | .NET 项目的根命名空间 | `MyProject` |

**命名规则**:
- 以字母开头
- 只能包含字母、数字、下划线
- 建议使用 PascalCase

### 后端配置

#### 数据库

| 选项 | 说明 | 适用场景 |
|------|------|----------|
| SQLite | 轻量级文件数据库 | 开发、原型、小型应用 |
| MySQL | 开源关系型数据库 | 中大型生产应用 |
| SQL Server | 微软企业级数据库 | 企业环境 |

#### 缓存

| 选项 | 说明 | 适用场景 |
|------|------|----------|
| 无 | 不启用缓存 | 简单应用 |
| 内存缓存 | 进程内缓存 | 单实例部署 |
| Redis | 分布式缓存 | 多实例部署 |

#### 功能模块

| 选项 | 说明 |
|------|------|
| JWT 认证 | 启用 JWT Token 认证 |
| Swagger 文档 | 启用 API 文档页面 |

### 前端配置

#### 路由模式

| 选项 | 说明 | URL 示例 |
|------|------|----------|
| Hash 模式 | URL 包含 `#` | `http://example.com/#/users` |
| History 模式 | 干净的 URL | `http://example.com/users` |

**注意**: History 模式需要服务器配置支持。

#### 其他选项

| 选项 | 说明 |
|------|------|
| 启用 Mock 数据 | 生成模拟数据，无需后端即可开发 |

## 生成项目

1. 填写配置选项
2. 右侧实时预览生成的文件结构
3. 点击「生成项目」按钮
4. 等待生成完成，自动下载 ZIP 文件

## 使用生成的项目

### 解压并进入项目

```bash
unzip MyProject.zip
cd MyProject
```

### 启动后端

```bash
cd src/MyProject.Api
dotnet restore
dotnet run
```

后端默认运行在 `http://localhost:5000`

### 启动前端

```bash
cd src/frontend
pnpm install
pnpm dev
```

前端默认运行在 `http://localhost:5173`

## 常见问题

### Q: 生成失败怎么办？

检查以下几点：
1. 项目名称是否符合命名规则
2. 网络连接是否正常
3. 查看浏览器控制台错误信息

### Q: 如何修改生成的代码？

生成的代码是完整的项目，可以直接编辑：
- 后端代码在 `src/[ProjectName].Api/` 目录
- 前端代码在 `src/frontend/` 目录

### Q: 支持哪些 IDE？

推荐使用：
- **后端**: Visual Studio 2022 / JetBrains Rider / VS Code
- **前端**: VS Code / WebStorm
